# ToUnicode Mapping Diagnostic Test

## Проблема

При объединении шрифтов наблюдается искажение текста:
- Оригинал: "All Rates Parallel Ladders as of 2025-10-17"
- Новый визуально: "ll ates Parallel Ladders as o 10 1"
- При копировании даты: "Ϯ0ϮϱͲ10Ͳ" вместо "2025-10-17"

Это классическая проблема с ToUnicode маппингом при слиянии шрифтов.

## Решение - Диагностический тест

Добавлен тест `DiagnoseTextTransformationIssue()` который отслеживает трансформацию ToUnicode mapping на всех этапах процесса объединения шрифтов.

### Структура теста

#### 1. Начальная диагностика (BEFORE MERGE)
Метод `DiagnoseToUnicodeMappings()` проверяет ToUnicode mapping в исходных шрифтах:
- Проверяет критические символы: A, R, 2, 5, -
- Извлекает CMap из ToUnicode stream
- Ищет маппинги Unicode → CID для каждого символа
- Отмечает отсутствующие символы (⚠️)

#### 2. Диагностика после слияния (AFTER MERGE)
Метод `DiagnoseToUnicodeMappingsInGlobalDict()` проверяет объединенные шрифты:
- Анализирует каждый merged font в global dictionary
- Проверяет сохранились ли ToUnicode CMaps
- Проверяет те же критические символы
- Выявляет потерянные или искаженные маппинги

#### 3. Анализ текста
Метод `AnalyzeTextExtraction()` проверяет финальный результат:
- Извлекает текст из обработанного PDF
- Сравнивает с ожидаемой фразой
- Показывает посимвольные различия с Unicode кодами
- Помогает точно определить какие символы искажены

## Ожидаемый результат

Тест покажет:
1. ✅ Какие символы присутствуют в ToUnicode до слияния
2. ⚠️ Какие символы отсутствуют в ToUnicode до слияния
3. ✅/⚠️ Какие символы сохранились/потерялись после слияния
4. ❌ Неправильные маппинги (например, R→Z)
5. 🔍 Точные Unicode коды искаженных символов

## Анализ результатов

### Если в консоли видите "NOT FOUND ⚠️"
Символ отсутствует в ToUnicode CMap - это основная причина проблемы.

### Если видите несовпадение CID
Символы маппятся на неправильные CID, что приводит к отображению других глифов.

### Если ToUnicode отсутствует после merge
Проблема в методе `CreateMergedFontDictionary()` - ToUnicode не копируется или копируется неправильный.

## Дальнейшие действия

После выявления конкретной причины через диагностику:

1. **Отсутствующий ToUnicode**: Убедиться что `toUnicode` копируется из правильного источника
2. **Неправильные маппинги**: Проверить логику слияния CMap
3. **Потерянные символы**: Может потребоваться объединение нескольких ToUnicode CMap
4. **Конфликтующие CID**: Нужно ремапить CID при слиянии шрифтов

## Критические символы для тестирования

- **A-Z**: Заглавные буквы (особенно 'A', 'R')
- **0-9**: Цифры (особенно '2', '5', '0', '1')
- **-**: Дефис/минус
- Специальные символы: пробелы, знаки пунктуации

Эти символы часто используются в заголовках и датах, где проблема наиболее заметна.
